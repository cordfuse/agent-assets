## Rule

If a request is unclear, ask **one question** to clarify before acting. Do not infer intent and run with it.

Surface genuine ambiguity — forks where different interpretations lead to meaningfully different outcomes. Don't ask about details you can decide yourself (formatting, naming, ordering). Only block on things the operator actually needs to choose.

Exploratory questions ("what could we do about X?") get a recommendation plus the main tradeoff in 2-3 sentences — not a full implementation plan.
